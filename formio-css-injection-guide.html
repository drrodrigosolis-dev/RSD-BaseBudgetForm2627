Form.io CSS Injection Quick Reference (Comprehensive)

This guide is designed for restricted Form.io licenses where you style forms by injecting CSS via an HTML component (or by attaching classes and targeting them from CSS). It includes a selector map, full component coverage, state styling, and copy-paste snippets.

1) Scope And Injection Points

In restricted setups, this is usually the safest pattern: place one HTML component near the top of the form with a <style> block, and scope every rule to a unique wrapper class or ID so styles do not leak to other forms. 

<!-- Form.io HTML component content -->
<style>
/* Scope to one form wrapper class or id */
#onboardingForm .form-control {
  border-radius: 10px;
}

#onboardingForm .btn-primary {
  background: #0f4fa8;
  border-color: #0f4fa8;
}
</style>
Recommended workflow: add a custom class in Layout/HTML attributes (example: onboarding-theme) and target .onboarding-theme ... selectors from the injected stylesheet.
Sanitization can remove style tags or attributes in some deployments. Always verify in the final runtime app (not only in builder preview).

Form.io DOM differs by renderer version/theme (Bootstrap 3/4/5). Treat these selectors as the practical superset and adjust with devtools.

3) Starter Scoped CSS Template

Use this as your baseline in the HTML component style block:

<style>
/* 1) Scope */
#myFormTheme {
  --fio-font: "Segoe UI", sans-serif;
  --fio-primary: #1e56b7;
  --fio-border: #c9d4ea;
  --fio-radius: 8px;
  --fio-danger: #c1263b;
}

/* 2) Base text and spacing */
#myFormTheme .formio-component,
#myFormTheme .form-group {
  margin-bottom: 12px;
}

#myFormTheme .control-label,
#myFormTheme .form-label {
  font: 600 0.93rem/1.3 var(--fio-font);
  margin-bottom: 6px;
}

/* 3) Inputs */
#myFormTheme .form-control {
  border: 1px solid var(--fio-border);
  border-radius: var(--fio-radius);
  padding: 8px 10px;
}

#myFormTheme .form-control:focus {
  border-color: var(--fio-primary);
  box-shadow: 0 0 0 3px rgba(30, 86, 183, 0.2);
  outline: none;
}

/* 4) Validation */
#myFormTheme .has-error .form-control,
#myFormTheme .form-control.is-invalid {
  border-color: var(--fio-danger);
  background: #fff5f7;
}

#myFormTheme .invalid-feedback,
#myFormTheme .help-block.text-danger {
  color: var(--fio-danger);
}

/* 5) Buttons */
#myFormTheme .btn-primary {
  background: var(--fio-primary);
  border-color: var(--fio-primary);
  color: #fff;
}
</style>

6) Atomic CSS Property Reference (A-Z Dictionary)

This section is fully disaggregated and alphabetized. Each row isolates one property with explicit syntax and a single example.

6.1 Fast Syntax For Surface And Effect Properties

/* Gradient */
background-image: linear-gradient(<angle>, <color-stop-1>, <color-stop-2>, ...);
background-image: radial-gradient(<shape> at <x y>, <color-stop-1>, <color-stop-2>, ...);

/* Blur/filter (on the element itself) */
filter: blur(<radius>);
filter: blur(1.2px) brightness(1.03) contrast(1.05);

/* Box shadow */
box-shadow: <offset-x> <offset-y> <blur-radius> <spread-radius> <color>;
box-shadow: 0 10px 22px 0 rgba(25,52,102,0.18);

/* Display */
display: block | inline | inline-block | flex | grid | none;

/* Backdrop filter (needs transparency to be visible) */
background: rgba(255,255,255,0.42);
backdrop-filter: blur(8px) saturate(130%);
-webkit-backdrop-filter: blur(8px) saturate(130%);

/* Animation shorthand */
animation: <name> <duration> <timing-function> <delay> <iteration-count> <direction> <fill-mode>;
animation: fioPulse 1.2s ease-in-out 0s infinite alternate both;

8) Specificity / !important / Sanitization

Specificity Strategy

  1. Always scope first: #myFormTheme .form-control over plain .form-control.
  2. Prefer component-specific selectors: .formio-component-email .form-control.
  3. Use !important only for third-party inline styles or unavoidable renderer conflicts.

Safe !important Example

#myFormTheme .form-control[readonly] {
  background: #eff4ff !important;
}

Sanitization / Runtime Validation

  • Confirm that <style> survives save + render + deployment pipeline.
  • If stripped, fallback to external app-level stylesheet or class-only hooks.
  • Test builder preview and embedded runtime separately; DOM wrappers may differ.
Avoid global selectors like input { ... } unless they are scoped. Unscoped CSS is the most common cause of cross-form breakage.

9) QA Checklist

  1. Desktop + mobile widths tested (at least 360px and 1280px).
  2. Focus visibility tested with keyboard only (Tab/Shift+Tab).
  3. Error states tested with required + invalid values.
  4. Disabled and read-only state legibility checked.
  5. DataGrid/EditGrid add/remove controls visually consistent.
  6. Tabs/Wizard active state clearly distinguishable.
  7. Color contrast validated for text/buttons/errors.
  8. No CSS leakage to other forms/pages from missing scope.

Tip: keep a small "theme token" section at the top of your style block and avoid hard-coded colors spread across rules.
