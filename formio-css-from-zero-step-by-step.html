Form.io CSS From Zero: Step-by-Step Guide

This guide assumes no prior CSS/Form.io experience. Follow it in order and you will create a stylesheet, inject it safely into Form.io, and apply it to your form.

What You Will Build

  1. A scoped stylesheet that only affects one form.
  2. A Form.io HTML component that injects that stylesheet.
  3. Optional HTML Attributes class hooks for targeted styling.
  4. A repeatable testing checklist for preview and production.

Step 1) Prepare Your Form Wrapper Scope

Goal

Create one unique class or ID to avoid styling all forms by accident.

Action

  1. Open your form in Form.io Builder.
  2. Find the top-level layout/container (or first parent container).
  3. In Layout -> HTML Attributes, add: class="onboarding-theme".
<!-- Scope target you will use in CSS -->
.onboarding-theme .form-control { ... }
If your environment supports IDs for the top wrapper, you can use id="onboardingTheme" and target #onboardingTheme .... Do not use both unless intentional.

Step 2) Create A Starter CSS Stylesheet

Goal

Start with a safe baseline that affects labels, inputs, errors, and buttons.

Action

  1. Create a local text file named formio-theme.css (optional but recommended).
  2. Paste this starter CSS.
/* Scope everything */
.onboarding-theme {
  --fio-primary: #1f5fbd;
  --fio-border: #c7d4ee;
  --fio-danger: #bd1f37;
}

.onboarding-theme .formio-component,
.onboarding-theme .form-group {
  margin-bottom: 12px;
}

.onboarding-theme .control-label,
.onboarding-theme .form-label {
  font-weight: 600;
  margin-bottom: 6px;
  color: #23365f;
}

.onboarding-theme .form-control {
  border: 1px solid var(--fio-border);
  border-radius: 9px;
  padding: 8px 10px;
}

.onboarding-theme .form-control:focus {
  border-color: var(--fio-primary);
  box-shadow: 0 0 0 3px rgba(31,95,189,.20);
  outline: none;
}

.onboarding-theme .has-error .form-control,
.onboarding-theme .form-control.is-invalid {
  border-color: var(--fio-danger);
  background: #fff5f7;
}

.onboarding-theme .invalid-feedback,
.onboarding-theme .help-block.text-danger {
  color: var(--fio-danger);
}

.onboarding-theme .btn-primary {
  background: var(--fio-primary);
  border-color: var(--fio-primary);
}

Step 3) Inject The CSS In Form.io

Goal

Load your CSS into the form using an HTML component.

Action

  1. Add an HTML component near the top of the form.
  2. Open the component settings.
  3. In the content area, wrap your CSS in a <style> tag.
<style>
.onboarding-theme .form-control {
  border-radius: 9px;
}

.onboarding-theme .btn-primary {
  background: #1f5fbd;
  border-color: #1f5fbd;
}
</style>
Keep one HTML component dedicated to CSS. This makes maintenance much easier.

Step 4) Add Optional Component-Level Hooks (HTML Attributes)

Goal

Tag specific components so you can style only those elements.

Action

  1. Open a specific component (example: Email field).
  2. Go to Layout -> HTML Attributes.
  3. Add class: class="field-email-main".
  4. Use CSS selector: .onboarding-theme .field-email-main .form-control { ... }.
/* Only this field */
.onboarding-theme .field-email-main .form-control {
  background: #f7fbff;
  border-color: #9fc2ff;
}

Step 5) Add Visual Effects (Optional)

Goal

Apply gradients, blur, shadows, display layouts, backdrop-filter, and animations correctly.

/* Gradient */
.onboarding-theme .panel {
  background-image: linear-gradient(120deg, #f7fbff, #e7f0ff);
}

/* Box shadow */
.onboarding-theme .panel {
  box-shadow: 0 10px 22px 0 rgba(25,52,102,.14);
}

/* Filter blur on element */
.onboarding-theme .some-preview {
  filter: blur(1.2px);
}

/* Backdrop filter (needs transparency) */
.onboarding-theme .glass {
  background: rgba(255,255,255,.45);
  backdrop-filter: blur(8px) saturate(130%);
  -webkit-backdrop-filter: blur(8px) saturate(130%);
}

/* Display layout */
.onboarding-theme .row-tools {
  display: flex;
  gap: 8px;
  align-items: center;
}

/* Animation */
.onboarding-theme .attention {
  animation: pulse 1.2s ease-in-out infinite;
}

@keyframes pulse {
  0%, 100% { transform: scale(1); }
  50% { transform: scale(1.06); }
}
Animations and pseudo-classes cannot be fully managed with inline styles alone. Use class hooks + injected CSS.

Step 6) Test In The Correct Order

Action Checklist

  • Builder preview looks correct.
  • Embedded/runtime app also looks correct.
  • Required field errors still visible.
  • Keyboard focus ring visible (Tab key).
  • Mobile width tested (around 360px).
  • Buttons and DataGrid actions still clickable.
  • No style leaks into other forms.
  • Disabled and read-only states are legible.

Step 7) Troubleshooting

Problem: CSS does nothing

  1. Confirm the HTML component actually contains <style>...</style>.
  2. Confirm your wrapper class exists in rendered HTML.
  3. Increase specificity: .onboarding-theme .formio-component-email .form-control.
  4. Use !important only for hard conflicts.

Problem: Backdrop blur not visible

  1. Ensure background uses alpha (example rgba(..., .45)).
  2. Ensure there is visual content behind the element.

Problem: Animations not running

  1. Verify matching @keyframes name exists.
  2. Check if prefers-reduced-motion media query disables motion.

Production Template (Copy-Paste)

<style>
/* 1) Scope */
.onboarding-theme {
  --fio-primary: #1f5fbd;
  --fio-border: #c7d4ee;
  --fio-danger: #bd1f37;
}

/* 2) Inputs */
.onboarding-theme .form-control {
  border: 1px solid var(--fio-border);
  border-radius: 9px;
  padding: 8px 10px;
}

.onboarding-theme .form-control:focus {
  border-color: var(--fio-primary);
  box-shadow: 0 0 0 3px rgba(31,95,189,.20);
  outline: none;
}

/* 3) Errors */
.onboarding-theme .has-error .form-control,
.onboarding-theme .form-control.is-invalid {
  border-color: var(--fio-danger);
}

/* 4) Buttons */
.onboarding-theme .btn-primary {
  background: var(--fio-primary);
  border-color: var(--fio-primary);
}

/* 5) Accessibility: reduce motion */
@media (prefers-reduced-motion: reduce) {
  .onboarding-theme * {
    animation: none !important;
    transition: none !important;
  }
}
</style>

Recommended process: update local CSS first, then paste final version into the HTML component.
